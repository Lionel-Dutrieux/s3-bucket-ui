# next-intl (en/fr) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Subagents run on the **sonnet** model (user requirement — never fable).

**Goal:** Toute string UI hardcodée passe par next-intl (anglais = référence, français complet), locale par cookie sans routing i18n.

**Architecture:** Mode next-intl « without i18n routing » : `getRequestConfig` lit le cookie `locale`, sinon négocie `Accept-Language`, défaut `en`. Pas de middleware, pas de segment `[locale]`. Messages en JSON racine (`messages/en.json`, `messages/fr.json`), un namespace par zone. Sélecteur de langue dans le menu utilisateur via server action.

**Tech Stack:** next-intl v4, Next.js 16 App Router (version modifiée — lire `node_modules/next/dist/docs/` en cas de doute), TanStack Form, zod, Biome, Vitest.

## Global Constraints

- Spec : `docs/superpowers/specs/2026-07-17-next-intl-design.md`.
- Hors périmètre : messages zod, emails nodemailer, logs serveur, README.
- `src/features/*` n'importe jamais d'une autre feature ni de `app/` (Biome l'enforce).
- Modules purs (`features/*/lib/`, `src/lib/`) : pas de `getTranslations` ni React — ils exposent des **clés**, la résolution se fait au point de rendu.
- Server actions : messages d'erreur traduits via `getTranslations` (de `next-intl/server`), retour `ActionResult` inchangé.
- Vérif par tâche : `pnpm typecheck && pnpm lint` (0 erreur). `pnpm test` quand la tâche touche des tests. `pnpm build` aux tâches 1 et finale.
- Chaque clé ajoutée existe dans **les deux** fichiers `messages/en.json` et `messages/fr.json` (même structure — le typage sur `en.json` casse sinon côté usage, pas côté fr : vérifier manuellement la symétrie).
- Français : orthographe et accents corrects, ton sobre (« Supprimer », pas « Suppression ! »).
- Anglais existant : reprendre les libellés actuels tels quels comme valeurs `en`.
- ICU pour toute interpolation/pluriel : `{name}`, `{count, plural, one {# file} other {# files}}`. Jamais de concaténation.
- Ne PAS traduire : noms de providers (« Amazon S3 »…), valeurs techniques (mime types, extensions), `aria-hidden`, données venant de la DB.
- Commits fréquents, un par tâche minimum.

## Pattern de remplacement (référence pour toutes les tâches de sweep)

Client component :

```tsx
"use client";
import { useTranslations } from "next-intl";

export function DeleteButton({ count }: { count: number }) {
  const t = useTranslations("browser");
  return <Button>{t("delete", { count })}</Button>;
  // en.json → "browser": { "delete": "Delete {count, plural, one {# item} other {# items}}" }
}
```

RSC / `generateMetadata` / server action / route handler :

```tsx
import { getTranslations } from "next-intl/server";

export async function generateMetadata() {
  const t = await getTranslations("admin");
  return { title: t("usersTitle") };
}
```

Server action (erreur traduite) :

```ts
"use server";
import { getTranslations } from "next-intl/server";

export async function renameEntry(input: unknown): Promise<ActionResult> {
  const t = await getTranslations("browser.errors");
  const parsed = schema.safeParse(input);
  if (!parsed.success) return { ok: false, error: t("invalidInput") };
  // …
}
```

Module pur (pas d'I/O) — exposer des clés, résoudre au rendu :

```ts
// features/activity/lib/operation-labels.ts (pur)
export const operationLabelKeys = { upload: "upload", delete: "delete" } as const;
// composant : const t = useTranslations("activity.operations"); t(operationLabelKeys[op])
```

Checklist par fichier de sweep : textes JSX, props `title`/`placeholder`/`aria-label`/`alt`/`label`, appels `toast.*(...)`, strings de `confirm-dialog`, valeurs par défaut de props, `generateMetadata`. Les clés en camelCase, groupées par sous-objet si le fichier est gros (`browser.shareDialog.title`).

`src/i18n/negotiation.ts` :

```ts
import { defaultLocale, isLocale, type Locale } from "./config";

/** Picks the best supported locale from an Accept-Language header. */
export function pickLocale(acceptLanguage: string | null): Locale {
  if (!acceptLanguage) return defaultLocale;
  const ranked = acceptLanguage
    .split(",")
    .map((part) => {
      const [tag, ...params] = part.trim().split(";");
      const q = params
        .map((p) => p.trim())
        .find((p) => p.startsWith("q="));
      return { tag: tag.toLowerCase(), q: q ? Number.parseFloat(q.slice(2)) : 1 };
    })
    .filter((e) => e.tag && !Number.isNaN(e.q))
    .sort((a, b) => b.q - a.q);
  for (const { tag } of ranked) {
    const base = tag.split("-")[0];
    if (isLocale(base)) return base;
  }
  return defaultLocale;
}
```

- [ ] **Step 5: Request config + messages squelette + provider**

`src/i18n/request.ts` :

```ts
import { getRequestConfig } from "next-intl/server";
import { cookies, headers } from "next/headers";
import { isLocale } from "./config";
import { pickLocale } from "./negotiation";

export default getRequestConfig(async () => {
  const store = await cookies();
  const cookieLocale = store.get("locale")?.value;
  const locale =
    cookieLocale && isLocale(cookieLocale)
      ? cookieLocale
      : pickLocale((await headers()).get("accept-language"));

  return {
    locale,
    messages: (await import(`../../messages/${locale}.json`)).default,
  };
});
```

`src/features/auth/actions.ts` :

```ts
"use server";

import { cookies } from "next/headers";
import { LOCALE_COOKIE } from "@/i18n/config";
import type { ActionResult } from "@/lib/action-result";
import { localeSchema } from "./lib/locale-schema";

export async function setLocale(input: unknown): Promise<ActionResult> {
  const parsed = localeSchema.safeParse(input);
  if (!parsed.success) return { ok: false, error: "Invalid locale" };
  const store = await cookies();
  store.set(LOCALE_COOKIE, parsed.data.locale, {
    path: "/",
    maxAge: 60 * 60 * 24 * 365,
    sameSite: "lax",
  });
  return { ok: true, data: undefined };
}
```
